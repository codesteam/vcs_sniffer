What is VCS sniffer?

The VCS sniffer lets you have check all your code before commit into VCS system (hg or git). It can restrict changes if your code is not valid. It is designed to make programming easier by making automation coding standard check. VCS sniffer require VCS and python only.

Supported sniffs

Specifically, we support checks for dynamic languages. More specific support checks is provided below.

  • PHP syntax check.
  • PHP coding standard check (we use PHPCodeSniffer).
  • ruby syntax check.
  • ruby coding standard check (we use rubocop).

hg extension

How install?

You can use VSC_sniffer as hg extension. For example, if you were to run

$ mkdir ~/.hgext/vcs_sniffer
$ cd ~/.hgext/vcs_sniffer
$ git clone git@github.com:codesteam/vcs_sniffer.git .

And then add this lines into your hgrc config:

[extensions]
vcs_sniffer = ~/.hgext/vcs_sniffer/vcs_sniffer

Sniff command

When the extension is properly installed, you can run command hg sniff to check all files in current repository folder.

Configuration

If you want change default VCS sniffer settings just add vcs_sniffer.yaml into repository root. VCS sniffer use PyYAML for parsing yaml files. You must install it before custom configuration usage: 

pip install pyyaml

See config example below:

utf_bom: true                                            # Check UTF encoding in file
php:                                                     # PHP specified settings
  extensions:                                            # List of PHP files extensions
    - php
    - php4
    - php5
  check_syntax: php -l %file%                            # PHP syntax check command
  check_cs: phpcs --standard=zend --report=emacs %file%  # PHP coding standard check cmd
  ignore:                                                # Folders to ignore
    - vendors
